Introduction to C & C Features

What is a Computer Programming Language (CPL)?

A CPL is an artificial language used to control the behaviour of a machine. Like human languages, CPLs are defined through syntactic (structure) and semantic (meaning) rules. They allow us to give instructions to a computer in a language it understands and are used to implement algorithms.

Levels of Programming Language

LevelDescriptionExamples
Low LevelBinary codes, CPU executes directly. Programmer's responsibility is more.Machine Language
Middle LevelOffers basic data structures and arrays; programmers handle operations.C, C++
High LevelProgrammer concentrates on algorithm; runtime handles details.Java, Python, Pascal

Programming Paradigms

  • Imperative – Explicit sequence of commands that update state. Example: Python
  • Declarative – Specify the result, not how to get it. Example: SQL, LISP
  • Structured – Clean, goto-free, nested control structures. Example: C Language
  • Procedural – Imperative with procedure calls. Example: C Language
  • Functional – Function calls that avoid global state. Example: Haskell, Scheme
  • Object-Oriented – Objects with encapsulated state send messages. Example: Java, C++, Python
  • Event-Driven – Emitters and listeners of asynchronous actions.
  • Logic/Rule-based – Facts and rules; an engine infers answers.

Why Learn C?

  • C gives importance to efficiency — does exactly what the programmer asks, nothing more.
  • C is not very safe; safety is the programmer's responsibility.
  • C is the language of choice for operating systems, compilers, linkers, loaders, device drivers.
  • C is consistently ranked among the top languages on the TIOBE Index.

Brief History of C

  • 1960s: Martin Richards developed BCPL
  • Ken Thompson enhanced it into B
  • 1969–1973: Dennis Ritchie developed C at Bell Labs, closely tied to Unix OS
  • Standardized: ANSI C (C89) → C99 → C11 → C17 → C2x

Key Features of C

  • Simple & Efficient – Basic syntax, easy to learn, widely used as an introductory language.
  • Fast – Statically typed and compiler-based (unlike interpreted Python/Java).
  • Portable – Machine-independent; code runs on various systems with minimal changes.
  • Extensibility – Easily extend existing programs with new features.
  • Function-Rich Libraries – Extensive built-in functions; user-defined functions can be added to libraries.
  • Dynamic Memory Management – malloc(), calloc(), realloc(), free().
  • Modularity with Structured Language – Break code into functions stored in libraries for reuse.
  • Mid-Level Language – Supports both low-level hardware manipulation and high-level programming.
  • Pointers – Directly interact with memory, arrays, functions, and structures.
  • Recursion – Functions can call themselves; enables code reusability and backtracking.

Applications of C

  • Operating Systems – Unix, Microsoft Windows, Android, Symbian
  • GUI – Adobe Photoshop, Adobe Illustrator, Adobe Premiere
  • Embedded Systems – Device drivers, microcontroller programming
  • Financial Systems – Bloomberg real-time financial information
Introduction to GCC & Program Development Life Cycle

GNU C Compiler (GCC) was originally authored by Richard Stallman, founder of the GNU Project (started 1984). First released in 1987 as the first portable ANSI C optimizing compiler released as free software. "GCC" now stands for GNU Compiler Collection, supporting C, C++, Fortran, ADA, Java, Objective-C, and more.

Features of GCC

  • Portable — runs on most platforms, produces output for many processor types
  • Supports microcontrollers, DSPs, and 64-bit CPUs
  • Can cross-compile — produce executables for a different system than the one running GCC
  • Modular design — new languages and architectures can be added
  • Free software under GNU GPL

Identifiers & Keywords

Tokens in C

A token is any identifier, keyword, variable, constant, string literal, or symbol. For example, printf("HelloFriends"); consists of 5 tokens: printf, (, "HelloFriends", ), ;

Identifier Rules

  • Starts with a letter (A–Z, a–z) or underscore (_)
  • Followed by zero or more letters, underscores, or digits (0–9)
  • Cannot use punctuation like #, @, %
  • Case-sensitive: Manpower ≠ manpower
  • Should not exceed 31 characters (some compilers)

Looping Constructs

while Loop — Pre-tested (0 or more times)

while — Various decrement versions
#include<stdio.h>
int main()
{
    int n = 5;
    while(n) { printf("%d ", n); n--; }          // 5 4 3 2 1
    n=5; while(n) printf("%d ", n--);             // 5 4 3 2 1
    n=5; while(n) printf("%d ", --n);             // 4 3 2 1 0
    n=5; while(n--) printf("%d ", n);             // 4 3 2 1 0
    n=5; while(--n) printf("%d ", n);             // 4 3 2 1
    return 0;
}

Reverse Bits

Arrays — Initialization & Traversal

What is a Data Structure?

Data structures store data in an organized and efficient manner so that retrieval is efficient. In C they are classified as Primitive (int, char, float, pointer) and Non-Primitive (Linear: Static→Array, Dynamic→Linked List/Stack/Queue; Non-Linear: Tree, Graph).

What is an Array?

An array is a linear data structure — a finite collection of similar (homogeneous) data items stored in successive (contiguous) memory locations. Arrays store multiple values in a single variable; they are fixed-size sequential collections of elements of the same type.

Properties of Arrays

Non-primary / secondary data type
Memory allocation is contiguous
Elements need not be unique
All elements must be same type (homogeneous)
Random access via index possible
Index (subscript) starts from 0
Memory allocated at compile time
Size is fixed at compile time
Cannot change size at runtime
Arrays are assignment incompatible
Out-of-bound access → undefined behaviour
sizeof(array) returns total bytes

1D Array — Declaration & Forms of Initialization

Address of a[i] = Base_Address + (sizeof(element) × i)
All Forms of Initialization
#include<stdio.h>
int main()
{
    // Syntax: Data_type Array_name[Size];
    int a1[10];         // 10×4 = 40 bytes; indices 0–9

    // 1. Full init — size inferred from initializer list
    int a2[] = {44,33,10,25,78,33,89};
    printf("sizeof a2 = %d\n", (int)sizeof(a2)); // 28

    // 2. Partial init — remaining filled with 0
    int a3[10] = {44,33,10};  // a3[3]…a3[9] = 0

    // 3. Designated Initializers (C99) — specify by index
    int a[15] = {[2]=29, [9]=7, [14]=48};
    printf("a[2]=%d a[9]=%d a[14]=%d\n", a[2], a[9], a[14]);

    // 4. Mixed (old + designated)
    int c[10] = {5, 1, 9, [4]=3, 7, 2, [8]=6};

    // 5. Sum of array elements
    int arr[] = {-1,4,3,1,7};
    int n = sizeof(arr)/sizeof(arr[0]);
    int sum = 0;
    for(int i=0; i<n; i++) sum += arr[i];
    printf("Sum = %d\n", sum); // 14
    return 0;
}
Undefined Behaviour
int marks[4] = {67,87,56,77,5}; — more initializers than size causes UB. Variable Length Arrays (VLAs) are unsafe — they can cause stack overflow.
2D Arrays — Multi-dimensional

A 2D array is an array of arrays. C uses row major ordering — all elements of one row are stored before all elements of the next row.

Addr(A[i][j]) = Base + ((i × cols) + j) × sizeof(element)
2D Array — Declaration, Init & Pointer Notation
#include<stdio.h>
int main()
{
    // Initialization with nested braces
    int marks[3][3] = {{90,95,91},{95,90,92},{99,91,93}};

    // Column size ALWAYS required when declaring without init
    // int c[][2] = {11,22,33,44};  // ✓ row inferred = 2
    // int d[][] = {...};             // ✗ compile error

    // Print matrix
    for(int i=0; i<3; i++) {
        for(int j=0; j<3; j++) printf("%d\t", marks[i][j]);
        printf("\n");
    }

    // Pointer notation for 2D arrays
    int a[5][2] = {{11,22},{33,44},{55,66},{77,88},{99,100}};
    printf("%d\n", a[3][1]);           // 88 — array notation
    printf("%d\n", *(*(a+3)+1));      // 88 — pointer notation
    int (*p)[2] = a;                  // pointer to a row (2 ints)
    printf("%d\n", p[3][1]);           // 88
    return 0;
}
Pointers

A pointer is a variable that stores the memory address of another variable. A pointer of a type can only point to that type. Size of any pointer is system-dependent (e.g., 8 bytes on 64-bit). A NULL pointer has value 0.

* in declaration → creates pointer. * outside declaration → dereferences (accesses the value at that address).

Pointer Declaration & Basics
#include<stdio.h>
int main()
{
    int *p;
    int a = 100;
    p = &a;
    printf("a = %d\n", a);    // 100
    printf("*p = %d\n", *p);  // 100 — dereference
    printf("p  = %p\n", p);   // address of a
    int arr[] = {12,44,22,33,55};
    int *p1 = arr; // same as p1 = &arr[0]
    printf("arr[2] via pointer: %d\n", *(p1+2)); // 22
    return 0;
}
Pointer Arithmetic

Valid operations: add int to pointer, subtract int from pointer, difference of two pointers (same array). The amount of change per ++ depends on the type pointed to.

*p++ vs (*p)++ vs *++p vs ++*p
#include<stdio.h>
int main()
{
    int arr[] = {12,33,44};
    int *p = arr;
    printf("%d\n", *p);   // 12
    p++;                    // address += sizeof(int)
    printf("%d\n", *p);   // 33
    // Expression table:
    // *p++    — use *p, then move pointer forward
    // *++p    — move pointer forward first, then dereference
    // (*p)++  — use value at p, then increment that value
    // ++*p    — increment value at p, return new value
    int a = 10; p = &a;
    printf("(*p)+1 = %d\n", (*p)+1); // 11 (p unchanged)
    printf("++*p   = %d\n", ++*p);   // 11 (value incremented)
    return 0;
}
Arrays vs Pointers — Array Traversal
AspectArrayPointer
sizeofTotal bytes of all elementsSize of pointer variable only
&nameAlias for &array[0]Address of the pointer variable
Assignmenta = p; ✗ NOT allowedp = a; ✓ allowed
Incrementa++; ✗ constant pointerp++; ✓ allowed
Equivalence
a[i], *(a+i), p[i], *(p+i), i[a], i[p] are ALL equivalent. An array is a constant pointer at compile time but degenerates to a pointer at runtime.
7 Versions of Array Traversal via Pointers
#include<stdio.h>
int main()
{
    int arr[] = {12,44,22,33,55};
    int *p = arr;

    printf("V1 p[i]:     "); p=arr; for(int i=0;i<5;i++) printf("%d ",p[i]);      printf("\n");
    printf("V2 *(p+i):   "); p=arr; for(int i=0;i<5;i++) printf("%d ",*(p+i));    printf("\n");
    printf("V3 *p++:     "); p=arr; for(int i=0;i<5;i++) printf("%d ",*p++);      printf("\n");
    printf("V6 *p,p++:   "); p=arr; for(int i=0;i<5;i++,p++) printf("%d ",*p); printf("\n");
    // V7: *arr++ — COMPILE ERROR (arr is a constant pointer)
    return 0;
}

Activation Record (Stack Frame)
When a function is called, an activation record is pushed to the call stack containing: local variables of the callee, return address, location to store return value, parameters, temporary variables. It is deleted when the function returns.
Parameter Passing in C
C Always Passes by Value
Arguments are copied to parameters. Changing the parameter inside the function does NOT affect the original argument. To modify the original, pass a pointer.
Pass by Value vs Pointer · Swap Correct vs Wrong
#include<stdio.h>

void fun_val(int a) { a = 200; }           // original unchanged
void fun_ptr(int *a) { *a = 200; }          // original IS changed
void swap_wrong(int a, int b) { int t=a; a=b; b=t; } // ✗ local only
void swap(int *a, int *b) { int t=*a; *a=*b; *b=t; }  // ✓ correct

int main()
{
    int a = 100;
    fun_val(a);   printf("After val: %d\n", a); // 100
    fun_ptr(&a); printf("After ptr: %d\n", a); // 200
    int x=100, y=200;
    swap(&x, &y);
    printf("x=%d y=%d\n", x, y); // 200 100
    return 0;
}
Dangling Pointer
Never return a pointer to a local variable. The local variable's memory is freed when the function returns. Dereferencing a dangling pointer is undefined behaviour.

void Data Type

void is an empty type with no value. Uses: (1) function return type when nothing is returned, (2) void *p — generic pointer not tied to any type (must be cast before dereferencing), (3) int f(void) — function takes no arguments.

Arrays & Functions

When an array is passed as a function argument, it degenerates to a pointer at runtime. sizeof(array) inside the function returns the pointer size, not the array size!

Read / Display / Sum via Functions (Best Practice)
#include<stdio.h>
void read_array(int[], int);
void display_array(const int*, int);
int  find_sum(const int*, int);

int main()
{
    int a[100], n;
    printf("How many elements? "); scanf("%d", &n);
    printf("sizeof a in main: %d\n", (int)sizeof(a)); // 400
    read_array(a, n);
    display_array(a, n);
    printf("Sum = %d\n", find_sum(a, n));
    return 0;
}
void read_array(int a[], int n) {
    printf("sizeof a in fn: %d\n", (int)sizeof(a)); // 8 (pointer!)
    for(int i=0; i<n; i++) scanf("%d", &a[i]);
}
void display_array(const int *a, int n) {
    for(int i=0; i<n; i++) printf("%d\t", a[i]); printf("\n");
}
int find_sum(const int *a, int n) {
    int s=0; for(int i=0; i<n; i++) s+=a[i]; return s;
}

Register Variables
register int i = 10; hints the compiler to store the variable in a CPU register. You cannot take the address of a register variable: &i → compile error. But you CAN store a pointer in a register: register int *a = &j;
Callbacks & Function Pointers

A callback (call-after function) is a function passed as an argument to another function to be called at a later time. In C, callbacks are implemented via function pointers. A function pointer stores the start address of executable code — it points to code, not data.

Function Pointer Syntax
int (*ptrFunc)(int, int); — ptrFunc is a pointer to a function that takes two ints and returns an int. Without parentheses around *ptrFunc, int *ptrFunc(int,int) would mean a function that returns int*.
Callback with add & mul
#include<stdio.h>

int what(int x, int y, int z, int (*op)(int,int,int)) {
    return op(x, y, z);
}
int add(int x, int y, int z) { return x+y+z; }
int mul(int x, int y, int z) { return x*y*z; }

int main()
{
    int (*p)(int,int,int);
    p = add; printf("%d\n", p(2,3,4));  // 9
    p = mul; printf("%d\n", p(2,3,4));  // 24
    printf("sum=%d\n",     what(1,3,4,add)); // 8
    printf("product=%d\n", what(5,3,4,mul)); // 60
    return 0;
}
Recursion

A function calling itself (directly or indirectly) with a termination/base condition is called recursion. Each recursive call pushes a new activation record to the call stack.

Rules for Recursion
1. Always define a base condition to stop recursion. 2. Each recursive call must move towards the base condition. 3. Without a base condition → infinite recursion → stack overflow.
Factorial — Iterative vs Recursive
#include<stdio.h>
int fact_iter(int n) {
    int r=1; for(int i=1; i<=n; i++) r*=i; return r;
}
int fact_rec(int n) {
    if(n==0) return 1;    // base condition
    return n * fact_rec(n-1);
}
int main() {
    for(int i=0; i<=7; i++)
        printf("%d! iter=%d rec=%d\n", i, fact_iter(i), fact_rec(i));
    return 0;
}
/* Trace: fact_rec(5) = 5*4*3*2*1*fact_rec(0) = 120 */

Searching Algorithms

Linear Search (Sequential Search)

Checks each element sequentially until a match is found or the whole array is traversed. Returns the index or -1 if not found. Works on unsorted or sorted arrays.

Linear Search
#include<stdio.h>
int linear_search(int *a, int n, int key) {
    for(int i=0; i<n; i++)
        if(a[i]==key) return i;
    return -1;
}
int main() {
    int a[] = {22,55,77,99,25,90,100}; int n=7;
    int r = linear_search(a, n, 100);
    r==-1 ? printf("Not found\n") : printf("Found at index %d\n", r);
    return 0;
}

Binary Search

Efficiently searches a sorted array by repeatedly halving the search space. Key must be comparable and array must be sorted.

  1. Compare the search key with the middle element.
  2. If match → return the middle index.
  3. If key < middle → search the left half (High = Mid − 1).
  4. If key > middle → search the right half (Low = Mid + 1).
  5. Repeat until found or Low > High (not found, return -1).
Binary Search — Iterative + Recursive
#include<stdio.h>
int bsearch_iter(int a[], int lo, int hi, int key) {
    while(lo<=hi) {
        int mid=(lo+hi)/2;
        if(a[mid]==key) return mid;
        else if(key<a[mid]) hi=mid-1;
        else               lo=mid+1;
    }
    return -1;
}
int bsearch_rec(int a[], int lo, int hi, int key) {
    if(lo>hi) return -1;
    int mid=(lo+hi)/2;
    if(a[mid]==key) return mid;
    else if(key<a[mid]) return bsearch_rec(a, lo, mid-1, key);
    else                  return bsearch_rec(a, mid+1, hi, key);
}
int main() {
    int arr[] = {2,3,5,7,8,10,12,15,18,20}; int n=10;
    printf("7  at index: %d\n", bsearch_iter(arr,0,n-1,7));   // 3
    printf("15 at index: %d\n", bsearch_rec(arr,0,n-1,15));  // 7
    printf("1  at index: %d\n", bsearch_iter(arr,0,n-1,1));   // -1
    return 0;
}

Selection Sort

Repeatedly finds the minimum element from the unsorted portion and places it at the beginning. Maintains two sub-arrays: sorted (left) and unsorted (right). n−1 passes needed.

  1. Start with position 0 as the assumed minimum position (pos = 0).
  2. Scan the rest of the array to find the actual minimum, update pos.
  3. Swap a[pos] with a[i] (if pos ≠ i).
  4. Advance i. Sorted portion grows by 1 each pass. Repeat n−1 times.
Selection Sort
#include<stdio.h>
void swap(int *x, int *y) { int t=*x; *x=*y; *y=t; }

void sort(int *a, int n) {
    for(int i=0; i<n-1; i++) {
        int pos=i;
        for(int j=i+1; j<n; j++)
            if(a[j]<a[pos]) pos=j;
        if(pos!=i) swap(&a[pos], &a[i]);
    }
}
int main() {
    int a[] = {22,19,14,25,99,81,7,45};
    int n = sizeof(a)/sizeof(*a);
    printf("Before: "); for(int i=0;i<n;i++) printf("%d ",a[i]); printf("\n");
    sort(a, n);
    printf("After:  "); for(int i=0;i<n;i++) printf("%d ",a[i]); printf("\n");
    return 0;
}
// Before: 22 19 14 25 99 81 7 45
// After:   7 14 19 22 25 45 81 99

find_frequency
#include<stdio.h>
int find_frequency(char *s, char ch) {
    int count = 0;
    for(; *s; s++)      // exits when *s == '\0'
        if(*s == ch) count++;
    return count;
}
