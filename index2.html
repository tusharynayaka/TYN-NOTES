Arrays — Initialization & Traversal

Topic 1 of 8 · CO1, CO2

What is a Data Structure?

Data structures store data in an organized and efficient manner so that retrieval is efficient. In C they are classified as Primitive (int, char, float, pointer, real, boolean) and Non-Primitive (Linear: Static→Array, Dynamic→Linked List/Stack/Queue; Non-Linear: Tree, Graph).

What is an Array?

An array is a linear data structure — a finite collection of similar (homogeneous) data items stored in successive (contiguous) memory locations. Instead of declaring 1000 separate variables for 1000 student marks, use a single array.

Key Definition
Arrays store multiple values in a single variable. They are fixed-size sequential collections of elements of the same type. The array name can be thought of as a collection of variables of the same type.

Properties of Arrays

Non-primary / secondary data type
Memory allocation is contiguous
Elements need not be unique
All elements must be same type (homogeneous)
Random access via index possible
Index (subscript) starts from 0
Memory allocated at compile time
Size is fixed at compile time
Cannot change size at runtime
Arrays are assignment incompatible
Out-of-bound access → undefined behaviour
sizeof(array) returns total bytes

Types of Arrays

CategoryTypes
By SizeFixed Length Array (compile-time size) · Variable Length Array (runtime size, unsafe)
By DimensionOne-Dimensional (Linear) · Multi-Dimensional

1D Array — Declaration

C Array Declaration Syntax 
// Syntax: Data_type Array_name[Size];
int    a1[10];      // 10 ints  → 10×4 = 40 bytes
float  average[6]; // 6 floats, indices 0 to 5
char   name[20];   // 20 chars, indices 0 to 19
double x[15];      // 15 doubles, indices 0 to 14

// Address of ith element:
// Addr(a[i]) = Base_Address + (size_of_element × i)
printf("size = %d\n", sizeof(a1)); // 40
Address of a[i] = Base_Address + (sizeof(element) × i)

Array Initialization

C All forms of initialization 
// 1. Full initialization — size inferred from elements
int a2[] = {44, 33, 10, 25, 78, 33, 89};
printf("%d\n", sizeof(a2)); // 7×4 = 28

// 2. Partial initialization — remaining filled with 0
int a3[10] = {44, 33, 10, 25, 78, 33, 89};
// a3[7], a3[8], a3[9] are automatically 0

// 3. Designated Initializers (C99) — specify by index
int a[15] = {[2] = 29, [9] = 7, [14] = 48};
// Equivalent — order does NOT matter:
int b[15] = {[14] = 48, [9] = 7, [2] = 29};

// 4. Mixed (old + designated)
int c[10] = {5, 1, 9, [4] = 3, 7, 2, [8] = 6};

// 5. Runtime initialization
int a5[5];
for(int i = 0; i < 5; i++)
    scanf("%d", &a5[i]);
⚠️ Undefined Behaviour: int marks[4] = {67,87,56,77,5}; — more initializers than size declared causes UB. Variable Length Arrays (VLAs) are unsafe — they can cause stack overflow.

Array Traversal

Accessing each element of the array is called traversal. Elements are accessed via the [] index operator.

C Coding Example 1 — Read & Display 
#include <stdio.h>

int main() {
    int arr[100];
    int n;

    printf("Enter number of elements: ");
    scanf("%d", &n);

    printf("Enter %d elements:\n", n);
    int i = 0;
    while(i < n) {
        scanf("%d", &arr[i]);
        i++;
    }

    printf("Entered elements:\n");
    for(i = 0; i < n; i++)
        printf("%d\n", arr[i]);

    return 0;
}
C Coding Example 2 — Sum of Array Elements 
int arr[] = {-1, 4, 3, 1, 7};
int sum = 0;
int n = sizeof(arr) / sizeof(arr[0]);

for(int i = 0; i < n; i++)
    sum += arr[i];

printf("Sum = %d\n", sum); // 14

2D Arrays — Multi-dimensional

A 2D array is an array of arrays. Every element is itself an array. Perfect for storing tabular data (e.g., marks of 5 students in 3 subjects).

C 2D Array — Declaration & Initialization 
// Declaration: data_type name[rows][cols]
int arr[4][2]; // 4 rows × 2 cols = 8 contiguous ints

// Initialization with nested braces
int marks[5][3] = {
    {90, 95, 91},
    {95, 90, 92},
    {99, 91, 93},
    {100, 99, 95},
    {89, 94, 100}
};

// Flat initialization — same result
int b[3][2] = {11, 22, 33, 44, 55, 66};

// Column size ALWAYS required:
int c[][2] = {11,22,33,44}; // ✓ row inferred = 2
// int d[][] = {...};             // ✗ compile error
Row Major Ordering (C default)
All elements of one row are stored followed by all elements of the next row. C uses row major ordering.

Formula: Address of A[i][j] = Base + ((i × No. of columns) + j) × sizeof(element)
Addr(A[i][j]) = Base + ((i × cols) + j) × sizeof(element)
C 2D Array — Read & Display 
#include <stdio.h>

int main() {
    int a[100][100], n, m;
    printf("Enter rows and cols: ");
    scanf("%d %d", &n, &m);

    printf("Enter %d elements:\n", n*m);
    for(int i = 0; i < n; i++)
        for(int j = 0; j < m; j++)
            scanf("%d", &a[i][j]);

    printf("Matrix:\n");
    for(int i = 0; i < n; i++) {
        for(int j = 0; j < m; j++)
            printf("%d\t", a[i][j]);
        printf("\n");
    }
    return 0;
}

Pointers

Topic 2 of 8 · CO2

What is a Pointer?

A pointer is a variable that stores the memory address of another variable. Not all pointers contain an address — a NULL pointer has value 0.

Pointer of a type can only point to that type
Size of any pointer is constant (system-dependent, e.g., 8 bytes on 64-bit)
NULL pointer has value 0
* in declaration → creates pointer; outside declaration → dereference
C Pointer Declaration & Basics 
// Syntax: Data-type *name;
int *p;           // p points to an integer
int  a = 100;
p = &a;           // p holds address of a

printf("a = %d\n", a);   // 100
printf("*p = %d\n", *p); // 100 — dereference
printf("p = %p\n", p);   // address of a

// Pointer pointing to array
int arr[] = {12, 44, 22, 33, 55};
int *p1 = arr; // same as p1 = &arr[0]

Pointer Arithmetic

Valid operations: add int to pointer, subtract int from pointer, difference of two pointers (same array).

C Coding Example 3 & 4 — Pointer Arithmetic 
int arr[] = {12, 33, 44};
int *p2 = arr;
printf("%d\n", *p2);  // 12
p2++;               // address += sizeof(int) = 4 bytes
printf("%d\n", *p2);  // 33

// Type affects increment amount:
int *p, x = 20;
p = &x;
printf("int* +1 = %p\n",    (int*)p+1);    // +4 bytes
printf("char* +1 = %p\n",   (char*)p+1);   // +1 byte
printf("double* +1 = %p\n", (double*)p+1); // +8 bytes

Tricky Pointer Expressions

C Examples 5 & 6 — *p++ vs (*p)++ 
int *p, a = 10;
p = &a;

(*p)+1;     // 11 — value +1, p unchanged
*p++;       // returns *p (10), THEN increments p (address)
(*p)++;     // returns *p, THEN increments the VALUE at p
++*p;       // increments value first, then returns it
ExpressionEffect
*p++Use *p, then move pointer forward
*++pMove pointer forward first, then dereference
(*p)++Use value at p, then increment that value
++*pIncrement value at p, return new value

Array Traversal using Pointers — 7 Versions

C Coding Example 7 — All traversal versions 
int arr[] = {12, 44, 22, 33, 55};
int *p3 = arr;

// V1: Array notation on pointer — p3 not modified
for(int i=0; i<5; i++) printf("%d ", p3[i]);      // 12 44 22 33 55

// V2: Pointer notation — p3 not modified
for(int i=0; i<5; i++) printf("%d ", *(p3+i));    // 12 44 22 33 55

// V3: *p3++ — use p3, then increment
for(int i=0; i<5; i++) printf("%d ", *p3++);     // 12 44 22 33 55

// V4: *++p3 — increments first (skips first element!)
for(int i=0; i<5; i++) printf("%d ", *++p3);     // 44 22 33 55 UB!

// V5: (*p3)++ — increments VALUE, not address
for(int i=0; i<5; i++) printf("%d ", (*p3)++);   // 12 13 14 15 16

// V6: separate increment in for-header
for(int i=0; i<5; i++, p3++) printf("%d ", *p3); // 12 44 22 33 55

// V7: *arr++ — COMPILE ERROR (arr is a constant pointer)
// for(int i=0; i<5; i++) printf("%d ", *arr++); // ✗

sizeofTotal bytes of all elementsSize of pointer variable only
&nameAlias for &array[0]Address of the pointer variable

Functions in C

Topic 3 of 8 · CO2

Why Functions?

Improve readability of code
Improve reusability — write once, call many times
Easier debugging — errors traced to specific functions
Reduces code size — redundant statements replaced by calls

Types of Functions

TypeDescriptionExamples
Library FunctionsPre-defined in C libraries; just include headerprintf(), scanf(), strcat()
User-defined FunctionsWritten by developer for specific tasksAny function you write

Three Components of a Function

C Declaration · Definition · Call 
// 1. DECLARATION (prototype) — must end with semicolon
int sum(int a, int b);
void disp_hello();

// 2. DEFINITION
int sum(int a, int b) {
    return a + b;
}

void disp_hello() {
    printf("Hello Friends\n");
}

int decrement(int y) { return y - 1; }

// INVALID definitions:
// int fun1(int a, int) { }  // ✗ unnamed parameter
// int fun2(int a, b)   { }  // ✗ missing type for b

// 3. CALL
int main() {
    int x = 100, y = 10;
    int result = sum(x, y);       // 110
    printf("%d\n", result);
    printf("%d\n", sum(x+6, y)); // arguments can be expressions
    return 0;
}
Activation Record (Stack Frame)
When a function is called, an activation record is pushed to the call stack containing: local variables of the callee, return address, location to store return value, parameters, temporary variables. It is deleted when the function returns.

Parameter Passing in C

✅ C always passes by VALUE. Arguments are copied to parameters. Changing the parameter inside the function does NOT affect the original argument.
C Coding Example 2 — Pass by Value vs Pointer 
// Pass by value — original NOT changed
void fun1_val(int a1) { a1 = 200; }

// Pass by pointer — original IS changed via *a1 = 200
void fun1_ptr(int *a1) { *a1 = 200; }

int main() {
    int a1 = 100;
    fun1_val(a1);   printf("%d\n", a1); // 100 — unchanged
    fun1_ptr(&a1); printf("%d\n", a1); // 200 — changed!
    return 0;
}

Swap Function — Correct vs Incorrect

C Coding Example 4 — Swap 
// Version 1 — WRONG: local copies swapped, originals unchanged
void swap_wrong(int a, int b) {
    int temp = a; a = b; b = temp;  // only local copies!
}

// Version 2 — CORRECT: swap via pointers
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int a = 100, b = 200;
    swap(&a, &b);
    printf("a=%d b=%d\n", a, b); // a=200 b=100 ✓
    return 0;
}
⛔ Dangling Pointer: Never return a pointer to a local variable. The local variable's memory is freed when the function returns, leaving the pointer pointing to invalid memory. Dereferencing a dangling pointer is undefined behaviour.

void Data Type

void is an empty type with no value. Uses: (1) function return type when nothing is returned, (2) void *p — generic pointer not tied to any type (must be cast before dereferencing), (3) int f(void) — function takes no arguments.

Arrays & Functions

When an array is passed as a function argument, it degenerates to a pointer at runtime. All valid pointer operations apply in the function body.

C Array & Functions — Version 4 (Best Practice) 
#include <stdio.h>

void  read_array(int[], int);
void  display_array(const int*, int); // const prevents modification
int   find_sum(const int*, int);

int main() {
    int a[100], n;
    printf("How many elements? ");
    scanf("%d", &n);
    printf("sizeof a in main: %d\n", (int)sizeof(a)); // 400
    read_array(a, n);
    display_array(a, n);
    printf("Sum = %d\n", find_sum(a, n));
    return 0;
}

void read_array(int a[], int n) {
    printf("sizeof a in fn: %d\n", (int)sizeof(a)); // 8 (pointer size!)
    for(int i=0; i<n; i++) scanf("%d", &a[i]);
}

void display_array(const int *a, int n) {
    for(int i=0; i<n; i++) printf("%d\t", a[i]);
    printf("\n");
    // a[4] = 8989; // ✗ compile error — const prevents this
}

int find_sum(const int *a, int n) {
    int sum = 0;
    for(int i=0; i<n; i++) sum += a[i];
    return sum;
}
💡 Use const int *a for display/sum functions to prevent accidental modification of the array inside the function. The sizeof of the array parameter inside the function returns the pointer size, not the array size!

Passing 2D Array to Functions — 6 Cases

C 2D Array Function Signatures 
// Case 1: No col size → COMPILE ERROR
// void read(int[][], int m, int n); // ✗

// Case 2: Col size != declared col → Warning, but may work
void read(int[][3], int, int);

// Case 3: Col size = declared col → Fine

// Case 4: VLA style — column from param (need reorder)
void read(int m, int n, int a[][n]);

// Case 5: Forward declaration (C99 VLA with original order)
void read(int n; int[][n], int, int);

// Case 6: Pointer to array (best for runtime)
void read(int n; int(*)[n], int, int);

Callbacks & Function Pointers

Topic 5 of 8 · CO2

What is a Callback?

A callback (also called "call-after function") is a function passed as an argument to another function to be called at a later time. In C, callbacks are implemented via function pointers.

A function pointer stores the start address of executable code. Unlike data pointers, function pointers point to code, not data.

Function Pointer Syntax
int (*ptrFunc)(int, int);
ptrFunc is a pointer to a function that takes two ints and returns an int.

Note: Without parentheses around *ptrFunc, int *ptrFunc(int,int) would mean a function that returns int*.
C Coding Example 1 — Callback with add & mul 
#include <stdio.h>

int what(int x, int y, int z, int (*op)(int, int, int)) {
    return op(x, y, z); // calls whatever function op points to
}

int add(int x, int y, int z) { return x + y + z; }
int mul(int x, int y, int z) { return x * y * z; }

int main() {
    int (*p)(int, int, int); // declare function pointer

    p = add;              // p = &add; both equivalent
    printf("%d\n", p(2, 3, 4));  // 9 — calls add

    p = mul;
    printf("%d\n", p(2, 3, 4));  // 24 — calls mul

    // Pass function name directly as argument (callback)
    printf("sum = %d\n",     what(1, 3, 4, add)); // 8
    printf("product = %d\n", what(5, 3, 4, mul)); // 60
    return 0;
}

Recursion

Topic 6 of 8 · CO1

What is Recursion?

A function calling itself (directly or indirectly) with a termination/base condition is called recursion. The solution is expressed in terms of a smaller version of the same problem.

Rules for Recursion
1. Always define a base condition to stop recursion.
2. Each recursive call must move towards the base condition.
3. Without a base condition → infinite recursion → stack overflow.

Factorial — Iterative vs Recursive

C Coding Example 2 — Factorial 
#include <stdio.h>

// Iterative
int fact_iter(int n) {
    int result = 1;
    for(int i = 1; i <= n; i++) result *= i;
    return result;
}

// Recursive — Logic: n! = n × (n-1)!  base: 0! = 1
int fact_rec(int n) {
    if(n == 0) return 1;
    return n * fact_rec(n - 1);
}

int main() {
    printf("6! = %d\n", fact_rec(6)); // 720
    return 0;
}

/*  Trace for fact_rec(5):
    =5*fact_rec(4) = 5*4*fact_rec(3)
    = 5*4*3*fact_rec(2) = 5*4*3*2*fact_rec(1)
    = 5*4*3*2*1*fact_rec(0) = 5*4*3*2*1*1 = 120  */

Binary Representation (what1)

C Coding Example 3 — Binary of n 
// Prints binary representation of n
int what1(int n) {
    if(n == 0) return 0;
    return (n % 2) + 10 * what1(n / 2);
}
// what1(8) → 1000

Count Set Bits (what2) & Power (what3)

C Coding Examples 4 & 5 
// Count 1-bits in binary representation
int count_bits(int n) {
    if(n == 0) return 0;
    return (n & 1) + count_bits(n >> 1);
}
// count_bits(10) → 2  (1010 has two 1s)

// b to the power p
int power(int b, int p) {
    if(p == 0) return 1;
    return b * power(b, p - 1);
}
// power(5, 2) → 25

// Sum 1+2+...+n
int sum_n(int n) {
    if(n == 0) return 0;
    if(n < 0)  return -1;
    return n + sum_n(n - 1);
}
// sum_n(5) → 15
Activation Record Stack
Each recursive call pushes a new activation record (stack frame) containing: local variables, return value, parameters, return address. For factorial(n), n activation records are on the stack at peak depth. The records pop off as the recursion unwinds.

Searching Algorithms

Topic 7 of 8 · CO1

Linear Search (Sequential Search)

Checks each element of the array sequentially until a match is found or the whole array is traversed. Returns position (index) or -1 if not found. Works on unsorted or sorted arrays.

C Coding Example 1 — Linear Search 
#include <stdio.h>

int linear_search(int *a, int n, int key) {
    for(int i = 0; i < n; i++) {
        if(a[i] == key) return i;
    }
    return -1;
}

int main() {
    int a[] = {22, 55, 77, 99, 25, 90, 100};
    int n = 7;

    int res = linear_search(a, n, 100);
    if(res == -1) printf("Not found\n");
    else          printf("Found at index %d\n", res); // 6

    res = linear_search(a, n, 177);
    printf("%s\n", res == -1 ? "Not found" : "Found");
    return 0;
}

Binary Search

Efficiently searches a sorted array by repeatedly halving the search space. Time required to access each element must be constant.

  1. Compare the search key with the middle element.
  2. If match → return the middle index.
  3. If key < middle → search the left half (set High = Mid − 1).
  4. If key > middle → search the right half (set Low = Mid + 1).
  5. Repeat until found or Low > High (not found, return -1).
C Coding Example 2 — Binary Search (Iterative + Recursive) 
#include <stdio.h>

// Iterative
int bsearch_iter(int a[], int low, int high, int key) {
    while(low <= high) {
        int mid = (low + high) / 2;
        if(a[mid] == key) return mid;
        else if(key < a[mid]) high = mid - 1;
        else               low  = mid + 1;
    }
    return -1;
}

// Recursive
int bsearch_rec(int a[], int low, int high, int key) {
    if(low > high) return -1; // base condition
    int mid = (low + high) / 2;
    if(a[mid] == key) return mid;
    else if(key < a[mid]) return bsearch_rec(a, low, mid-1, key);
    else                   return bsearch_rec(a, mid+1, high, key);
}

int main() {
    int arr[] = {2, 3, 5, 7, 8, 10, 12, 15, 18, 20};
    int n = 10;

    int pos = bsearch_iter(arr, 0, n-1, 7);
    printf("7 at index: %d\n", pos); // 3

    pos = bsearch_rec(arr, 0, n-1, 15);
    printf("15 at index: %d\n", pos); // 7
    return 0;
}

Best CaseO(1) — first elementO(1) — middle element

Selection Sort

Repeatedly finds the minimum element from the unsorted portion and places it at the beginning. Maintains two sub-arrays: sorted (left) and unsorted (right).

  1. Start with position 0 as the assumed minimum position (pos = 0).
  2. Scan the rest of the array to find the actual minimum, update pos.
  3. Swap a[pos] with a[i] (if pos ≠ i).
  4. Advance i. Sorted portion grows by 1 each pass. Repeat n−1 times.
C Coding Example 1 — Selection Sort 
#include <stdio.h>

void swap(int *x, int *y) {
    int temp = *x; *x = *y; *y = temp;
}

void sort(int *a, int n) {
    for(int i = 0; i < n-1; i++) {
        int pos = i;  // assume current element is smallest
        for(int j = i+1; j < n; j++) {
            if(a[j] < a[pos]) pos = j; // update min position
        }
        if(pos != i) swap(&a[pos], &a[i]); // place min at i
    }
}

int main() {
    int a[] = {22, 19, 14, 25, 99, 81};
    int n = sizeof(a) / sizeof(*a);

    printf("Before: ");
    for(int i = 0; i < n; i++) printf("%d ", a[i]);

    sort(a, n);

    printf("\nAfter:  ");
    for(int i = 0; i < n; i++) printf("%d ", a[i]);
    printf("\n");

Solution — Problem 1: Character Count

C find_frequency — 3 versions 
#include <stdio.h>
#include <string.h>

int find_frequency(char *s, char ch) {
    int count = 0;
    for(; *s; s++)          // exits when *s == '\0'
        if(*s == ch) count++;
    return count;
}

int main() {
    char str[] = "whatsapp";
    char ch = 'a';
    printf("'%c' appears %d times in '%s'\n",
            ch, find_frequency(str, ch), str); // 3
    return 0;
}

Solution — Problem 6: Pair Addition

C pair_addition 
#include <stdio.h>

void pair_addition(int *a, int n) {
    int i;
    for(i = 0; i < n-1; i += 2)
        printf("%d\t", a[i] + a[i+1]);
    if(n % 2) printf("%d", a[i]); // leftover odd element
    printf("\n");
}

int main() {
    int a1[] = {2,3,4,5,6,7};
    int a2[] = {2,3,4,5,6};
    pair_addition(a1, 6); // 5 9 13
    pair_addition(a2, 5); // 5 9 6
    return 0;
}
