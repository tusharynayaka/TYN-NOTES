Introduction to C & C Features

What is a Computer Programming Language (CPL)?

A CPL is an artificial language used to control the behaviour of a machine. Like human languages, CPLs are defined through syntactic (structure) and semantic (meaning) rules. They allow us to give instructions to a computer in a language it understands and are used to implement algorithms.

Levels of Programming Language

LevelDescriptionExamples
Low LevelBinary codes, CPU executes directly. Programmer's responsibility is more.Machine Language
Middle LevelOffers basic data structures and arrays; programmers handle operations.C, C++
High LevelProgrammer concentrates on algorithm; runtime handles details.Java, Python, Pascal

Programming Paradigms

  • Imperative – Explicit sequence of commands that update state. Example: Python
  • Declarative – Specify the result, not how to get it. Example: SQL, LISP
  • Structured – Clean, goto-free, nested control structures. Example: C Language
  • Procedural – Imperative with procedure calls. Example: C Language
  • Functional – Function calls that avoid global state. Example: Haskell, Scheme
  • Object-Oriented – Objects with encapsulated state send messages. Example: Java, C++, Python
  • Event-Driven – Emitters and listeners of asynchronous actions.
  • Logic/Rule-based – Facts and rules; an engine infers answers.

Why Learn C?

  • C gives importance to efficiency — does exactly what the programmer asks, nothing more.
  • C is not very safe; safety is the programmer's responsibility.
  • C is the language of choice for operating systems, compilers, linkers, loaders, device drivers.
  • C is the language of choice for hardware-related software.
  • C is consistently ranked among the top languages on the TIOBE Index (updated monthly).

Brief History of C

  • 1960s: Martin Richards developed BCPL
  • Ken Thompson enhanced it into B
  • 1969–1973: Dennis Ritchie developed C at Bell Labs, closely tied to Unix OS
  • Standardized: ANSI C (C89) → C99 → C11 → C17 → C2x

Key Features of C

  • Simple & Efficient – Basic syntax, easy to learn, widely used as an introductory language.
  • Fast – Statically typed and compiler-based (unlike interpreted Python/Java).
  • Portable – Machine-independent; code runs on various systems with minimal changes.
  • Extensibility – Easily extend existing programs with new features.
  • Function-Rich Libraries – Extensive built-in functions; user-defined functions can be added to libraries.
  • Dynamic Memory Management – malloc(), calloc(), realloc(), free().
  • Modularity with Structured Language – Break code into functions stored in libraries for reuse.
  • Mid-Level Language – Supports both low-level hardware manipulation and high-level programming.
  • Pointers – Directly interact with memory, arrays, functions, and structures.
  • Recursion – Functions can call themselves; enables code reusability and backtracking.

Applications of C

  • Operating Systems – Unix (original OS written in C), Microsoft Windows, Android, Symbian
  • GUI – Adobe Photoshop, Adobe Illustrator, Adobe Premiere
  • Embedded Systems – Device drivers, microcontroller programming
  • Financial Systems – Bloomberg real-time financial information
Introduction to GCC & Program Development Life Cycle

GNU C Compiler (GCC) was originally authored by Richard Stallman, founder of the GNU Project (started 1984). First released in 1987 as the first portable ANSI C optimizing compiler released as free software. "GCC" now stands for GNU Compiler Collection, supporting C, C++, Fortran, ADA, Java, Objective-C, and more.

Features of GCC

  • Portable — runs on most platforms, produces output for many processor types
  • Supports microcontrollers, DSPs, and 64-bit CPUs
  • Can cross-compile — produce executables for a different system than the one running GCC
  • Modular design — new languages and architectures can be added
  • Free software under GNU GPL
Installation
Windows: Install via MinGW — Tutorial Link
Linux/macOS: GCC available by default

Program Development Life Cycle (PDLC)

Phases: Editing → Pre-processing → Compilation → Linking → Loading → Execution

1. Editing

Enter and save the source program. Example: create first.c

first.c
#include <stdio.h>
int main()
{
    // This is a comment — helps readability
    printf("Hello Friends");
    return 0;
}

2. Pre-Processing

Tasks performed: (a) Macro substitution, (b) Comments stripped off, (c) Expansion of included files. The output is called a translation unit.

Pre-processing command
# No file created — output on stdout
$ gcc -E first.c
Observation
The original comment disappears (stripped). The #include <stdio.h> is replaced by the entire content of stdio.h.

3. Compilation

Converts source code into machine code (1s and 0s). Output is an object file (.o).

Compile
$ gcc -c first.c   # Creates first.o (object file)

4. Linking

Combines one or more object files into one executable. Resolves external function definitions (e.g., printf). Output is a.out (Linux) or a.exe (Windows).

Linking
$ gcc first.o              # Creates a.out / a.exe
$ gcc first.o -o PESU    # Output file named PESU

5. Executing

Execute
# Windows
a.exe       # or  PESU

# Linux
./a.out     # or  ./PESU
Characteristics & Program Structure

Key Characteristics of a C Program

  • C is case-sensitive — Main ≠ main
  • Free format source code — indentation doesn't affect logic, but should be used for readability
  • Every program has a well-defined entry point: main()
  • Pre-processor directives start with #
  • Each statement ends with a semicolon (;) — acts as statement terminator
  • main returns 0 on success, non-zero on failure (by convention)

Warnings — Why Enable -Wall?

Warnings are diagnostic messages that report potentially erroneous but not immediately illegal code. Always compile with -Wall:

Common Warning-triggering Mistakes
Forgetting to initialize a variable · Forgetting to return a value from a function · printf/scanf format string mismatches · Function used without prior declaration

{

Identifiers & Keywords

Tokens in C

A token is any identifier, keyword, variable, constant, string literal, or symbol. For example, printf("HelloFriends"); consists of 5 tokens: printf, (, "HelloFriends", ), ;

Identifiers

A name used to identify a variable, function, or user-defined item. Rules:

  • Starts with a letter (A–Z, a–z) or underscore (_)
  • Followed by zero or more letters, underscores, or digits (0–9)
  • Cannot use punctuation like #, @, %
  • Case-sensitive: Manpower ≠ manpower
  • Should not exceed 31 characters (some compilers)

Valid examples: _sum, stud_name, a_123, myname50, count123

Keywords (32 in C)

Keywords
Keywords have special meaning in C and cannot be used as variable names. Examples: int, float, char, double, void, return, if, else, while, for, do, switch, case, break, continue, goto, struct, union, typedef, sizeof, extern, static, auto, register, const, volatile, signed, unsigned, short, long, enum, default
Variables & Data Types

Variables

A variable is a named storage area that programs can manipulate. It has a name, value, location, and type.

  • Declaration – Informs the compiler: name, type, optional initial value. In C, declaration and definition happen together (memory is allocated).
  • Definition – Memory is allocated. In C, same as declaration (except with extern).
  • Initialization – Assigning a value at the point of declaration. Uninitialized variables have undefined values.
Variable Examples
int c = 100;    // Declaration + initialization
int d;           // Undefined value — assign later
d = 300;         // Assignment (assignment is an expression in C)
// int c = 200;  → Error: cannot re-declare in same scope
extern int a;   // Declaration only, no memory — discussed in Unit 5

Input & Output Functions in C

Formatted Input — scanf

scanf syntax
int scanf(const char *format, ...);
  • Returns >0 — items converted/assigned successfully
  • Returns 0 — no item assigned
  • Returns <0 — read error or EOF before any assignment
Important Rules
Always use & (address operator) with primary type variables in scanf. Avoid escape sequences like \n inside scanf format strings — they cause scanf not to terminate properly.

Unformatted Input — getchar

Reading Characters

int main()
{
    char ch = 'p';
    printf("Ch is %c\n", ch);   // p

    /* Reading two chars
    char x = getchar();
    char y = getchar();
    putchar(x);
    putchar(y); */

    // If user enters P<newline>:
    // x = 'P', y = '\n'

scanf vs getchar
scanf / printf: Generic, handles multiple types. getchar / putchar: Not generic, reads/writes a single char — more efficient for character I/O.

Increment & Decrement Operators

  • Pre (++i) — increment first, then use the value
  • Post (i++) — use the value first, then increment
Pre vs Post Increment
#include<stdio.h>
int main()
{
    int a = 34, b;
    b = a++;   // b = 34, a becomes 35 (post)
    printf("b=%d a=%d\n", b, a);

    a = 34;
    b = ++a;   // a becomes 35, b = 35 (pre)
    printf("b=%d a=%d\n", b, a);
    return 0;
}

Sequence Points & Short-circuit Evaluation

Sequence points guarantee all side effects of prior evaluations are complete. Basic sequence points in C: &&, ||, ?:, ,, and the expression of a while loop.

Undefined Behavior Warning
a + a++ is undefined — the value of the left a is not defined relative to the increment.
b = a * a++ is undefined — value of leftmost a depends on compiler. Don't write this!
Short-circuit / Sequence Point Demo
#include<stdio.h>
int main()
{
    int a = 11;
    printf("%d\n", a++==12 || a==12);  // 1 (true)
    printf("%d\n", a);                    // 12
    // a++ returns 11 (false), but a becomes 12 at ||
    // Then a==12 is true → overall true
    return 0;
}

Looping Constructs

while Loop

Pre-tested loop — body executes 0 or more times. The expression is a sequence point.

while — Display n to 0 (various versions)
#include<stdio.h>
int main()
{
    int n = 5;

    // Version 3: Recommended — clear + indented + block
    while(n)
    {
        printf("%d ", n);   // 5 4 3 2 1
        n = n - 1;
    }

    // Version 4: Post-decrement in loop body
    n = 5;
    while(n) printf("%d ", n--);   // 5 4 3 2 1

    // Version 5: Pre-decrement in loop body
    n = 5;
    while(n) printf("%d ", --n);   // 4 3 2 1 0

    // Version 6: Post-decrement in condition (sequence point)
    n = 5;
    while(n--) printf("%d ", n);   // 4 3 2 1 0 (n already decremented)

    // Version 7: Pre-decrement in condition
    n = 5;
    while(--n) printf("%d ", n);   // 4 3 2 1 (n-1 iterations)

    return 0;
}

for Loop

for loop is isomorphic with while — same semantics, different style. Best when there is clear initialization, condition, and modification.

for — Sum 1 to n
#include<stdio.h>
int main()
{
    int n = 10;
    int sum = 0;
    // for(init; condition; modification)
    for(int i = 1; i <= n; ++i)
    {
        sum += i;   // sum = sum + i
    }
    printf("Sum = %d\n", sum);  // 55
    return 0;
}

do...while Loop

Bottom-tested loop — body always executes at least once. Useful when you need to perform the action first and then check the condition.

do-while — Digital Root
#include<stdio.h>
int main()
{
    int n = 493;
    int s;
    // Digital root: keep summing digits until single digit
    do
    {
        for(s = 0; n; n /= 10)
            s += n % 10;
        n = s;
    } while(s > 9);

    printf("Digital root = %d\n", s);
    return 0;
}
Selection Statements

if / if-else

Triangle Classification
#include<stdio.h>
int main()
{
    int a, b, c, count = 0;
    scanf("%d%d%d", &a, &b, &c);
    if(a == b) ++count;
    if(b == c) ++count;
    if(c == a) ++count;
    // count 0 = scalene, 3 = equilateral, 1 = isosceles

    switch(count)
    {
        case 0: printf("Scalene\n");       break;
        case 1: printf("Isosceles\n");     break;
        case 3: printf("Equilateral\n");   break;
        default: printf("Invalid\n");     break;
    }
    return 0;
}

switch Statement

Use when comparing an integer/char variable against constants with equality checks. The rule of C: "break if switch" — without break, execution falls through to next case.

Vowel Check using switch
#include<stdio.h>
int main()
{
    char ch = getchar();
    switch(ch)
    {
        case 'a': case 'A':
        case 'e': case 'E':
        case 'i':
        case 'o':
        case 'u':
            printf("Vowel\n"); break;
        default:
            printf("Not a vowel\n");
    }
    return 0;
}

Nested Control Structures — Armstrong Numbers

Armstrong Numbers 0–999
#include<stdio.h>
int main()
{
    int h, t, u, hc, tc, uc, sum, number;
    for(h = 0; h < 10; h++)
    {
        hc = h*h*h;  // cube of hundreds digit (computed once per h)
        for(t = 0; t < 10; t++)
        {
            tc = t*t*t;  // computed once per t
            for(u = 0; u < 10; u++)
            {
                uc = u*u*u;
                sum = hc + tc + uc;
                number = h*100 + t*10 + u;
                if(number == sum)
                    printf("%d\n", number);
            }
        }
    }
    return 0;
}
// Armstrong numbers: 0,1,153,370,371,407

Single Character I/O & Word Count Problem

Goal: Simulate the Unix wc command — count characters, words, and lines from user input or a file (using input redirection).

Input Redirection
$ ./a.out < filename — OS opens the file and makes it available as keyboard input to the program.
Reading till EOF
#include<stdio.h>
int main()
{
    // EOF is a macro in stdio.h with value -1
    char ch;
    while((ch = getchar()) != EOF)
        putchar(ch);   // Echo everything until EOF
    return 0;
}
Word Count — Basic Version
#include<stdio.h>
int main()
{
    char ch;
    int nl = 0, nw = 0, nc = 0;
    while((ch = getchar()) != EOF)
    {
        nc++;
        if(ch == '\n') nl++;
        if(ch == ' ' || ch == '\t' || ch == '\n') nw++;
    }
    printf("words=%d lines=%d chars=%d\n", nw, nl, nc);
    return 0;
}
// Issue: multiple spaces/newlines each increment word count
Word Count — Improved (with inword flag)
#include<stdio.h>
int main()
{
    int inword = 0, nl = 0, nw = 0, nc = 0;
    char ch;
    while((ch = getchar()) != EOF)
    {
        nc++;
        if(ch == '\n') nl++;
        if(inword && (ch==' ' || ch=='\n' || ch=='\t'))
        {
            nw++;
            inword = 0;
        }
        else if(!(ch==' ' || ch=='\n' || ch=='\t'))
            inword = 1;
    }
    printf("words=%d lines=%d chars=%d\n", nw, nl, nc);
    return 0;
}
// Fixed: multiple spaces/newlines don't inflate word count
Language Specifications & Behaviors

A language specification describes how a compiler/interpreter should react to various inputs. C standards define four types of behavior:

Behavior TypeDescriptionExample
Locale-specificDepends on C library implementationCharacter classification
UnspecifiedMultiple valid behaviors; not required to be consistentOrder of argument evaluation in printf
Implementation-definedBehavior documented by each compilerSize of int
UndefinedNo requirements — may crash, compile error, or wrong outputDivision by zero, array out-of-bounds
Undefined Behavior
The C standard imposes no requirements on undefined behavior. The program may fail to compile, crash, generate incorrect results, or coincidentally do what was intended. Never rely on UB!
C Standard Behavior Demo
#include<stdio.h>
int main()
{
    // printf("%d\n", 5/0);     // Undefined — division by zero

    // float a = 23.5;
    // printf("a is %d\n",a); // Undefined — wrong format spec

    // printf("%u %u %u\n", sizeof(int), sizeof(short int), sizeof(float));
    // → Implementation-defined behavior (sizes vary)

    int d = 10;
    printf("%d %d %d\n", d++, d++ - --d, --d);
    // Unspecified behavior — evaluation order of arguments not defined!
    return 0;
}
